# AI Light Novel Translator (Ollama Pipeline)

Локальный конвейер перевода корейских ранобэ на русский через Ollama.
Восемь стадий: нарезка → глоссарий → кросс-проверка глоссария → перевод →
кросс-проверка перевода → литературная вычитка → QA → финальный судья,
плюс экспорт. Опционально — облачный фолбэк через Gemini для добивания
сегментов, которые не вытянули локальные модели.

Модели (`config.py`): перевод/глоссарий — `qwen3:30b-a3b-instruct-2507-q4_K_M`,
независимая сверка — `aya-expanse:32b`, литературная полировка — `gemma3:27b`,
судья — `magistral:24b`.

## Установка

```bash
pip install -r requirements.txt
ollama serve
ollama pull qwen3:30b-a3b-instruct-2507-q4_K_M
ollama pull aya-expanse:32b
ollama pull gemma3:27b
ollama pull magistral:24b
```

`requirements.txt` подтягивает и `google-genai` — он нужен только для
`gemini_fallback.py`. Если облачный фолбэк не планируется, пакет можно не
устанавливать (или поставить отдельно: `pip install -q -U google-genai`).
Локальный пайплайн работает без него.

Положи исходный корейский текст в `input/source.txt`.

## Порядок запуска

Весь конвейер одной командой: **`run_all.bat`** (двойной клик). Он прогоняет все
стадии по очереди, после перевода и вычитки зовёт `gemini_fallback`, пишет полный
лог в `output/pipeline.log`, проверяет, что Ollama жива, и не падает, если облачный
фолбэк дал сбой. Скрипт **resume-friendly**: если прервать его на полпути
(`Ctrl+C`, выключение, ошибка), просто запусти `.bat` ещё раз — уже сделанные
стадии пропускаются, а длинные стадии (перевод/проверка/вычитка/судья) сами
продолжают с того сегмента, на котором остановились (прогресс сохраняется
посегментно). Rolling-бэкапы стадий складываются в `output/backups/`.

> Для НОВОЙ главы перед запуском **очисти или перемести папку `output/`** —
> иначе будут переиспользованы старые сегменты/глоссарий/перевод.

Или по шагам вручную:

```bash
# 1. Нарезка на сегменты
python stage1_split.py                      # regex (по умолчанию)
# python stage1_split.py --method kss       # точнее, нужен kss+pecab

# 2. Глоссарий (имена/термины + hanja + гоноративы)
python stage2_glossary.py

# 3. Кросс-проверка глоссария независимой моделью (aya) — ПРОВЕРЬ РУКАМИ после!
python stage3_glossary_check.py

# 4. Перевод с глоссарием и сквозной связностью (qwen)
python stage4_translate.py
python stage4_translate.py --resume
python stage4_translate.py --retry-errors

# 5. Кросс-проверка перевода независимой моделью (aya): смысл + выравнивание реплик
python stage5_translate_check.py
python stage5_translate_check.py --resume

# 6. Литературная вычитка (gemma)
python stage6_proofread.py --export-txt
python stage6_proofread.py --resume
python stage6_proofread.py --retry-fallbacks

# 7. QA — отчёт о проблемных местах (без обращения к модели)
python stage7_qa.py

# 8. Финальный судья (LLM-as-judge, Magistral) — оценивает и перечисляет проблемы
python stage8_judge.py

# Экспорт финального текста
python export_final.py --format plain        # только русский
python export_final.py --format with_source  # KO/RU абзацами
python export_final.py --format side_by_side # для сравнения
```

Старые скрипты (исходная версия с гитхаба) сохранены в `ver1.0/` как архив —
из коробки не запускается, пути в его `config.py` указывают внутрь `ver1.0/`.

### Облачный фолбэк (ручной): `gemini_fallback.py`

Когда локальные модели на части сегментов выдают мусор (остатки корейского,
мало кириллицы, петли повторов, `[ОШИБКА ПЕРЕВОДА]`) — добиваем облачным Gemini.
Ключ кладётся в `apikey.txt` рядом со скриптами (или env `GEMINI_API_KEY`).
Пакет: `pip install -q -U google-genai` (уже в `requirements.txt`).

```bash
python gemini_fallback.py --dry-run          # показать проблемные сегменты (без вызовов)
python gemini_fallback.py                     # перевести все автонайденные проблемные
python gemini_fallback.py --ids 6,25          # добить конкретные сегменты вручную
python gemini_fallback.py --ids 6 --also-final  # ещё и обновить 4_final.json
python gemini_fallback.py --mode polish --ids 14 # перешлифовать сегмент в 4_final.json
```

Исправленные сегменты помечаются `translated_by="gemini"` / `cloud_fallback=true`,
прежняя битая попытка сохраняется в `failed_translation`. Модель и путь к ключу —
`MODEL_CLOUD` / `APIKEY_FILE` в `config.py`. Запускать после stage5 (перед stage6)
или после stage7 по отчёту QA.

### Финальный судья (LLM-as-judge): `stage8_judge.py`

Оценивает финальный перевод (`4_final.json`) против корейского оригинала силами
Mistral Magistral Small 24B (`MODEL_JUDGE`, по умолчанию `magistral:24b`). Ставит
баллы (accuracy / fluency / terminology / completeness) и выдаёт список конкретных
проблем по серьёзности. **Ничего не меняет в переводе** — только судит. Magistral —
рассуждающая модель, её `<think>` скрипт срезает сам.

```bash
python stage8_judge.py                 # судить весь 4_final.json
python stage8_judge.py --resume        # продолжить, пропуская оценённые
python stage8_judge.py --only-flagged  # в txt только проблемные (verdict != ok)
python stage8_judge.py --dry-run       # сколько сегментов будет оценено
```

Результат: `output/6_judge_report.json` (полные данные) и `6_judge_report.txt`
(читаемый список проблем, отсортирован по серьёзности).

## Что изменилось относительно `ver1.0/` (архив с гитхаба)

**Архитектура**

- Стадий стало 8 вместо 4: добавлены независимая кросс-проверка глоссария
  (`stage3`), кросс-проверка перевода (`stage5`), мехQA (`stage7`) и
  LLM-судья (`stage8`).
- Добавлен **ручной облачный фолбэк** `gemini_fallback.py` для добивания того,
  что локальные модели не вытянули.
- Добавлен **`run_all.bat`** — resume-friendly прогон всего пайплайна с
  логом и rolling-бэкапами в `output/backups/`.
- Все модели обновлены: переводчик `qwen3:14b → qwen3:30b-a3b-instruct-2507-q4_K_M`,
  проверяльщик `aya-expanse:32b` (новый), полировщик `gemma3:27b`, судья
  `magistral:24b` (новый).

**Глоссарий (`stage2` / `stage3`)**

- **Поле `hanja` (漢字).** Для сино-корейских концепт-терминов модель сперва
  восстанавливает иероглифы и переводит ПО ИХ СМЫСЛУ, а не фонетикой. Это лечит
  у корня кальки вида «Гуин Цзюймэй» (надо: «меридианы Девяти Инь») и «Чхонму
  Чхи» (надо: «тело Небесного Воина»). Ханча показывается переводчику и
  проверяльщику в подсказке и используется как опора.
- **Гоноративы/обращения сеются принудительно** из `config.HONORIFICS`
  (`도련님 → молодой господин` и т.п.) как `builtin`-термины. Экстрактор намеренно
  выбрасывает «общие слова», поэтому раньше обращения угадывались фонетически
  («Домини»). `builtin` не правится кросс-проверкой.
- **Детерминированный слой:** `output/2_glossary_overrides.json` (если есть) —
  высший приоритет, переживает пере-прогон. Сюда складывай выверенные термины.
- `korean / romanization / hanja` НЕПРИКОСНОВЕННЫ на кросс-проверке (aya меняет
  только русскую сторону / категорию / заметку).

**Перевод (`stage4`)**

- **Сырой «ПРЕДЫДУЩИЙ ПЕРЕВОД» в промпте ВЫКЛЮЧЕН** по умолчанию
  (`TRANSLATE_USE_PREV_CONTEXT=False`). Именно он провоцировал галлюцинации-копии
  (в `ver1.0` в #25 продублировался диалог из #24). Связность теперь держит только
  абстрактное «бегущее саммари» (story bible).
- Параметры декодирования вынесены в config (`TRANSLATE_TOP_P/TOP_K/REPEAT_PENALTY/
  PRESENCE_PENALTY`). NB: на части сборок Ollama штрафы для Qwen могут молча
  игнорироваться — см. комментарий в config и проверку 1.0 vs 1.5.

**Кросс-проверка перевода (`stage5`)**

- **Выравнивание реплик.** Проверяльщик обязан сверить число реплик/предложений
  с корейским и УДАЛИТЬ строки, которых в оригинале нет (ловит галлюцинации-копии),
  а не только править лексику. В системный промпт добавлен few-shot ровно на
  случай #25.
- **`valid()` переякорен.** Корректное СОКРАЩЕНИЕ (удаление галлюцинации) больше
  не откатывается на черновик. Ограничен только рост относительно черновика
  (чтобы проверяльщик сам не дописал) и грубое укорочение относительно корейского
  источника (реальный пропуск).

**Вычитка (`stage6`, gemma) — без изменений логики:** показала себя хорошо;
работает только с формой, держится глоссария, длина в коридоре с откатом.

## Ключевые настройки (`config.py`)

| Параметр | Назначение |
| --- | --- |
| `MODEL_TRANSLATE`, `MODEL_CHECK`, `MODEL_PROOFREAD`, `MODEL_JUDGE` | модели Ollama |
| `MODEL_CLOUD`, `APIKEY_FILE` | облачный Gemini (фолбэк) |
| `HONORIFICS` | фиксированные обращения, всегда в глоссарии |
| `TRANSLATE_USE_PREV_CONTEXT` | сырой контекст пред. сегмента (по умолч. False) |
| `TRANSLATE_*_PENALTY`, `TRANSLATE_TOP_P/TOP_K` | декодирование переводчика |
| `SUMMARY_UPDATE_EVERY` | частота обновления бегущего саммари (0 = выкл) |
| `GLOSSARY_NORMALIZE_CHUNK` | размер чанка при нормализации/проверке глоссария |
| `GLOSSARY_OVERRIDES_FILE` | путь к ручным оверрайдам глоссария |
| `QA_*` | пороги флагов в стадии QA |
| `JUDGE_*` | контекст/температура/предикт-лимит судьи |
| `BACKUP_DIR` | папка для rolling-бэкапов (`output/backups/`) |
