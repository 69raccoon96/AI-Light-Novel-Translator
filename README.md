# AI Light Novel Translator (Ollama Pipeline)

```
Петпроект был создан в качестве эксперимента и был полностью написан нейросетями.
(и даже этот README)
```

Автоматизированный пайплайн для перевода корейских веб-новелл и ранобэ на русский язык с использованием локальных нейросетей через [Ollama](https://ollama.ai/).

Проект разбивает процесс на логические этапы: от сегментации текста до составления глоссария, перевода с учетом контекста и финальной литературной вычитки.

## 🌟 Особенности

- **Полностью локально:** Текст не отправляется на сторонние серверы (OpenAI, DeepL и т.д.), всё работает на вашем железе через Ollama.
- **Умный глоссарий:** Автоматическое извлечение имен, топонимов и уникальных терминов. Транслитерация по системе Концевича.
- **Контекстный перевод:** Модель переводит каждый сегмент, «видя» предыдущий переведенный текст и подтягивая только те термины из глоссария, которые есть в текущем абзаце.
- **Атомарное сохранение и чекпоинты:** Процесс можно прервать в любой момент. Скрипт продолжит работу с места остановки.
- **Защита от галлюцинаций:** Встроенные проверки (длина, соотношение кириллицы/хангыля), автоматический Fallback и возможность повторно перевести только сегменты с ошибками.

## 🛠 Требования

- **Python**: 3.10 или выше.
- **Ollama**: Установленная и запущенная локально (по умолчанию на `http://localhost:11434`).
- Желательно не менее 12 ГБ памяти видеокарты, чтобы комфортно запускать модели на 14B - 27B параметров.

### Рекомендуемые модели
- Для перевода и глоссария: **qwen3:14b-q4_K_M** (Идеально подходит для видеокарты с объемом памяти 12 Гб).
- Для литературной вычитки: **gemma3:27b** (Подходит для видеокарт с объемом памяти 12 Гб, если памяти больше - лучше выбрать модель посильнее)

## 🚀 Установка

1. Склонируйте репозиторий

2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
   *(Примечание: библиотеки `kss` и `pecab` нужны для более умной разбивки корейского текста на предложения. Если их не установить, скрипт будет использовать базовый Regex).*

3. Скачайте нужные модели в Ollama:
   ```bash
   ollama pull qwen3:14b-q4_K_M
   ollama pull gemma3:27b
   ```

## ⚙️ Настройка

Откройте файл `config.py` и настройте параметры под себя:
- `MODEL_GLOSSARY`, `MODEL_TRANSLATE`, `MODEL_PROOFREAD` — укажите названия скачанных моделей.
- Температуру генерации (`TRANSLATION_TEMPERATURE` и др.).
- Размер сегментов текста.

**Важно:** В файлах `stage2_glossary.py` и `stage3_translate.py` в промптах указаны примеры перевода имён (Юдер Байель и др.). **Обязательно замените их на примеры из вашей новеллы**, чтобы нейросеть лучше понимала желаемый стиль!

## 📖 Использование (Пошаговый гайд)

Создайте папку `input` в корне проекта и положите туда исходный корейский текст в формате `.txt` (файл `source.txt`).

### Этап 1: Разбивка текста
Разделяет сплошной текст на удобные для нейросети сегменты (по умолчанию до 1500 символов).
```bash
python stage1_split.py --method kss
```
*(Или `--method regex`, если не установлен `kss`).*

### Этап 2: Создание глоссария
Нейросеть читает текст батчами, извлекает имена и термины, а затем нормализует их (убирает дубликаты, проверяет пол и транслитерацию).
```bash
python stage2_glossary.py --batch-size 5
```
> ⚠️ **ВНИМАНИЕ:** Обязательно откройте файл `output/2_glossary.json` и проверьте термины вручную! Исправьте ошибки транслитерации, удалите мусор. Качество глоссария = качество всего перевода.  
> **Рекомендуется потратить несколько долларов и отдать глоссарий на вычитку облачным моделям (Например Gemini)**

### Этап 3: Черновой перевод
Модель переводит текст, строго придерживаясь глоссария и учитывая предыдущий контекст.
```bash
python stage3_translate.py
```
*Если перевод прервался, просто запустите команду с флагом `--resume`. Чтобы перевести заново сегменты с ошибками (fallback), используйте `--retry-errors`.*

### Этап 4: Литературная вычитка
Вторая модель (редактор) проходит по готовому тексту, исправляет грамматику, убирает кальки с корейского синтаксиса и делает текст живым.
```bash
python stage4_proofread.py --mode full
```
*(Доступен режим `--mode light` для минимальных пунктуационных правок).*

### Этап 5: Экспорт результата
Сборка JSON файлов в итоговый текстовый документ.
```bash
python export_final.py --format plain
```
Доступные форматы:
- `plain` — только русский текст.
- `with_source` — корейский оригинал абзаца и русский перевод под ним.
- `side_by_side` — разделено чертами для удобного сравнения.

## 📂 Структура проекта

```text
├── config.py             # Глобальные настройки путей, моделей и параметров
├── export_final.py       # Утилита экспорта JSON -> TXT
├── stage1_split.py       # Нарезка текста (regex / kss)
├── stage2_glossary.py    # Извлечение терминов (Ollama)
├── stage3_translate.py   # Контекстный перевод (Ollama)
├── stage4_proofread.py   # Литературная редактура (Ollama)
├── requirements.txt      # Зависимости Python
├── input/                # Папка для source.txt
└── output/               # Папка для сгенерированных JSON файлов
```
